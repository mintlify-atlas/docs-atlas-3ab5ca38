## Overview

The Books Management feature allows users to view and manage their book collection. Books are displayed in a scrollable list with options to view details and add new books to the library.

## Data Structure

Books in AppBiblioteca follow a simple interface structure:

```typescript
interface Book {
  id: number;
  title: string;
}
```

## Viewing Books List

The books list is the main interface for browsing your book collection. It's implemented using React Native's `FlatList` component for optimal performance with large datasets.

### Implementation

```typescript
import React, { useEffect, useState } from "react";
import { FlatList } from "react-native";
import api from "@/api/axios";

export default function Books() {
  const [books, setBooks] = useState<Book[]>([]);

  useEffect(() => {
    try {
      api.get("/books/").then((response) => {
        setBooks(response.data);
      });
    } catch (error) {
      console.log("Error fetching books =>", error);
    }
  }, []);

  return (
    <FlatList
      data={books}
      keyExtractor={(item) => item.id.toString()}
      renderItem={({ item }) => <Item title={item.title} />}
    />
  );
}
```

**Source:** `app/(main)/(tabs)/books.tsx:31-52`

### Key Features

- **Automatic Data Fetching**: Books are fetched automatically when the component mounts using `useEffect`
- **Optimized Rendering**: Uses `FlatList` for efficient rendering of large lists
- **Unique Keys**: Each book item uses its ID as a unique key for React reconciliation

## API Integration

Books are fetched from the backend API using the axios instance configured with authentication.

### Endpoint

```
GET /api/books/
```

### Response Format

```json
[
  {
    "id": 1,
    "title": "The Great Gatsby"
  },
  {
    "id": 2,
    "title": "To Kill a Mockingbird"
  }
]
```

### Authentication

All API requests automatically include the Bearer token from secure storage via axios interceptors.

## Error Handling

The books list implements basic error handling for API failures:

```typescript
try {
  api.get("/books/").then((response) => {
    setBooks(response.data);
  });
} catch (error) {
  console.log("Error fetching books =>", error);
}
```

Errors are logged to the console for debugging purposes. In production, you may want to:

- Display user-friendly error messages
- Implement retry logic
- Show loading states while fetching
- Handle offline scenarios
