## Overview

The Authors Management feature provides a comprehensive view of all authors in your library. Authors are displayed in a scrollable list with options to view their books and manage author information.

## Data Structure

Authors in AppBiblioteca use a simple, focused data structure:

```typescript
interface Author {
  id: number;
  name: string;
}
```

## Viewing Authors List

The authors list provides a clean, scrollable interface for browsing all authors in your library.

### Implementation

```typescript
import React, { useEffect, useState } from "react";
import {
  View,
  Text,
  FlatList,
  StyleSheet,
  StatusBar,
  TouchableOpacity,
} from "react-native";
import api from "@/api/axios";
import { SafeAreaProvider, SafeAreaView } from "react-native-safe-area-context";

export default function Authors() {
  const [authors, setAuthors] = useState<Author[]>([]);

  useEffect(() => {
    try {
      api.get("/authors/").then((response) => {
        setAuthors(response.data);
      });
    } catch (error) {
      console.log("Error fetching authors =>", error);
    }
  }, []);

  return (
    <SafeAreaProvider>
      <SafeAreaView style={styles.container}>
        <Text>Lista de autores</Text>
        <FlatList
          data={authors}
          keyExtractor={(item) => item.id.toString()}
          renderItem={({ item }) => <Item name={item.name} />}
        />
      </SafeAreaView>
    </SafeAreaProvider>
  );
}
```

## Performance Considerations

### FlatList Optimization

The component uses `FlatList` for optimal performance:

```typescript
<FlatList
  data={authors}
  keyExtractor={(item) => item.id.toString()}
  renderItem={({ item }) => <Item name={item.name} />}
/>
```

**Benefits:**

- **Virtualization**: Only renders visible items plus a small buffer
- **Memory Efficiency**: Recycles components as users scroll
- **Smooth Scrolling**: Optimized for large datasets
- **Unique Keys**: Uses author IDs for efficient re-rendering
