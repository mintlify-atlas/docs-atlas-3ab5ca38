## Overview

AppBiblioteca uses **React Native's StyleSheet API** for styling components. This provides a clean, performant way to style your React Native components with a CSS-like syntax.

## React Native StyleSheet

### Basic Usage

Create styles using `StyleSheet.create()` at the bottom of your component file:

```typescript
import { StyleSheet, View, Text } from 'react-native';

export default function MyComponent() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Hello World</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 20,
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
  },
});
```

## Flexbox Layout

React Native uses Flexbox for layouts. Here are common patterns:

### Vertical Layout (Default)

```typescript
const styles = StyleSheet.create({
  container: {
    flex: 1,
    // flexDirection: 'column' is the default
    justifyContent: 'center',    // Centers vertically
    alignItems: 'center',        // Centers horizontally
  },
});
```

### Horizontal Layout

```typescript
const styles = StyleSheet.create({
  row: {
    flexDirection: 'row',         // Horizontal layout
    justifyContent: 'space-between',  // Space items apart
    alignItems: 'center',         // Vertically center items
  },
});
```

### Common Flex Properties

<CodeGroup>

```typescript Flex Sizing
container: {
  flex: 1,  // Takes all available space
}

leftColumn: {
  flex: 2,  // Takes 2/3 of space
}

rightColumn: {
  flex: 1,  // Takes 1/3 of space
}
```

```typescript Justify Content
container: {
  justifyContent: 'flex-start',    // Top (vertical) / Left (horizontal)
  justifyContent: 'center',        // Center
  justifyContent: 'flex-end',      // Bottom (vertical) / Right (horizontal)
  justifyContent: 'space-between', // Space between items
  justifyContent: 'space-around',  // Space around items
}
```

```typescript Align Items
container: {
  alignItems: 'flex-start',  // Left (vertical) / Top (horizontal)
  alignItems: 'center',      // Center
  alignItems: 'flex-end',    // Right (vertical) / Bottom (horizontal)
  alignItems: 'stretch',     // Stretch to fill
}
```

</CodeGroup>

## SafeAreaView Usage

Use `SafeAreaView` to avoid notches, status bars, and other device-specific UI elements:

```typescript
import { SafeAreaProvider, SafeAreaView } from 'react-native-safe-area-context';

export default function MyScreen() {
  return (
    <SafeAreaProvider>
      <SafeAreaView style={styles.container}>
        {/* Your content */}
      </SafeAreaView>
    </SafeAreaProvider>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
  },
});
```

<Info>
`SafeAreaView` automatically adds padding to avoid unsafe areas like the notch on iPhone X and newer devices.
</Info>

## Responsive Design Patterns

### Using Dimensions

```typescript
import { Dimensions } from 'react-native';

const { width, height } = Dimensions.get('window');

const styles = StyleSheet.create({
  container: {
    width: width * 0.9,  // 90% of screen width
  },
  card: {
    width: width / 2 - 20,  // Half screen minus margins
  },
});
```

### Platform-Specific Styles

```typescript
import { Platform } from 'react-native';

const styles = StyleSheet.create({
  container: {
    padding: 20,
    ...Platform.select({
      ios: {
        shadowColor: '#000',
        shadowOffset: { width: 0, height: 2 },
        shadowOpacity: 0.1,
        shadowRadius: 4,
      },
      android: {
        elevation: 4,
      },
    }),
  },
});
```

## Combining Styles

### Multiple Styles

```typescript
<View style={[styles.container, styles.centered]}>
  <Text style={[styles.text, { color: 'red' }]}>Hello</Text>
</View>

const styles = StyleSheet.create({
  container: {
    flex: 1,
  },
  centered: {
    justifyContent: 'center',
    alignItems: 'center',
  },
  text: {
    fontSize: 16,
  },
});
```

### Conditional Styles

```typescript
<TouchableOpacity 
  style={[
    styles.button,
    isDisabled && styles.buttonDisabled
  ]}
>
  <Text>Press Me</Text>
</TouchableOpacity>
```

## Best Practices

<Steps>
  <Step title="Use StyleSheet.create()">
    Always create styles with `StyleSheet.create()` instead of inline objects for better performance.
    
    ```typescript
    // ✅ Good
    const styles = StyleSheet.create({ container: { flex: 1 } });
    <View style={styles.container} />
    
    // ❌ Avoid
    <View style={{ flex: 1 }} />
    ```
  </Step>

  <Step title="Keep Styles at the Bottom">
    Place StyleSheet definition at the bottom of the file for better readability.
  </Step>

  <Step title="Name Styles Semantically">
    Use descriptive names that indicate purpose, not appearance.
    
    ```typescript
    // ✅ Good
    styles.errorText
    styles.primaryButton
    
    // ❌ Avoid
    styles.redText
    styles.blueButton
    ```
  </Step>

  <Step title="Use Flex for Layouts">
    Prefer flexbox over absolute positioning for responsive layouts.
  </Step>

  <Step title="Add SafeAreaView">
    Wrap your screens in `SafeAreaView` to handle device-specific safe areas.
  </Step>

  <Step title="Centralize Colors">
    Create a constants file for colors and reuse them throughout the app.
  </Step>
</Steps>
